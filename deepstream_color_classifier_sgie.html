01

Pipeline Position

The color classifier runs as a Secondary GIE — after the tracker so it receives one inference call per tracked object crop per frame, not per raw detection. The SGIE's operate-on-gie-id must match the PGIE's gie-unique-id.

Why After Tracker? The tracker assigns stable object_id values. If the SGIE ran before the tracker each detection would be a new anonymous object — you could not accumulate per-track color history.

Reading SGIE Output in the Probe

SGIE classifier results land in NvDsClassifierMeta attached to each NvDsObjectMeta. The unique_component_id field identifies which SGIE produced the result — match it against your gie-unique-id=2.

Python · metadata traversal
def get_color_prediction(obj_meta):
    """
    Returns (label, confidence) from the color SGIE.
    Returns (None, 0.0) if SGIE result not yet available
    (e.g. object too small, async mode lag).
    """
    l_cls = obj_meta.classifier_meta_list
    while l_cls:
        cls_meta = pyds.NvDsClassifierMeta.cast(l_cls.data)

        # Filter to our color SGIE by unique_component_id
        if cls_meta.unique_component_id == 2:
            l_label = cls_meta.label_info_list
            while l_label:
                label_info = pyds.NvDsLabelInfo.cast(l_label.data)
                # result_label: string (e.g. "silver")
                # result_prob:  float  (softmax probability)
                return label_info.result_label, label_info.result_prob
                l_label = l_label.next

        l_cls = l_cls.next
    return None, 0.0
Async Mode With classifier-async-mode=0 results are attached to the same frame's metadata. With async mode enabled, results arrive on a subsequent frame — you need to handle the lag in your accumulator. Sync mode is simpler and recommended unless latency is critical.
05

The Stabilization Problem

Your classifier fires once per frame. Raw per-frame predictions are noisy because the same physical vehicle produces different image patches depending on viewpoint, lighting, and occlusion. You need to aggregate N predictions into one stable label.

Noise SourceEffect on Prediction
Lighting variationSilver → white in direct sun; gray → black in shadow
Viewpoint changeFront vs rear reflectance differs; specular highlights shift
Partial occlusionAnother vehicle overlaps bbox; wrong color dominates crop
Low bbox resolutionDistant vehicles → <64px → skipped by SGIE or low confidence
Motion blurColor information smeared across pixels; softmax flattens
Strategy A

Confidence-Weighted Vote

Accumulate (label, conf) pairs. At flush, sum confidence per label. Winner is stable color. Robust and simple.

Strategy B

EMA on Logits

Exponential moving average on raw probability vector. Requires tensor extraction. Best for gradual convergence as vehicle approaches.

Strategy C

Quality-Gated Vote

Weight each vote by both SGIE confidence and frame quality score. Most accurate. Combines signals from the best-frame pipeline.

06

Strategy A — Confidence-Weighted Majority Vote

The recommended default. No tensor extraction required — works entirely on result_label and result_prob from NvDsLabelInfo.

Python
from collections import defaultdict

def compute_stable_color(color_history):
    """
    color_history: list of (label: str, conf: float) tuples

    Returns: (stable_label: str, stability: float)
      stability ∈ [0, 1] — fraction of total weight on winner
    """
    if not color_history:
        return None, 0.0

    # Low-confidence frames add noise — filter first
    filtered = [(lbl, conf) for lbl, conf in color_history
                if conf > 0.40]

    # Fallback: use all frames if too few pass the gate
    if len(filtered) < 3:
        filtered = color_history

    # Accumulate confidence weight per class label
    weight_map = defaultdict(float)
    for label, conf in filtered:
        weight_map[label] += conf

    # Winner = highest cumulative weight
    stable_label  = max(weight_map, key=weight_map.get)
    total_weight  = sum(weight_map.values())
    winner_weight = weight_map[stable_label]

    # Stability: 1.0 = all votes on one label, 0.1 = spread evenly
    stability = winner_weight / total_weight if total_weight > 0 else 0.0

    return stable_label, stability
07

Strategy B — EMA on Softmax Logits

Requires raw probability vectors (full softmax output, not just top-1). Extract via NvDsInferTensorMeta or a custom output layer probe. Better than discrete voting when the vehicle approaches gradually — the EMA naturally increases confidence as more frames arrive.

Python
import numpy as np

@dataclass
class ColorEMA:
    n_classes: int   = 10
    alpha:     float = 0.3   # higher = more weight on recent frames
    ema:       np.ndarray = None
    n_frames:  int   = 0

    def update(self, probs: np.ndarray):
        """probs: shape (n_classes,), softmax output"""
        if self.ema is None:
            self.ema = probs.copy()
        else:
            self.ema = self.alpha * probs + (1 - self.alpha) * self.ema
        self.n_frames += 1

    def predict(self, min_frames: int = 5):
        """
        Returns (class_idx, confidence) or (None, 0.0)
        if insufficient observations.
        """
        if self.n_frames < min_frames or self.ema is None:
            return None, 0.0
        idx  = int(np.argmax(self.ema))
        conf = float(self.ema[idx])
        return idx, conf

# Per-track EMA state
color_ema_state = {}   # (src_id, oid) → ColorEMA

def update_ema(key, probs):
    if key not in color_ema_state:
        color_ema_state[key] = ColorEMA()
    color_ema_state[key].update(probs)
Alpha Tuning alpha=0.3 is a good default. Lower alpha (0.1–0.15) gives a longer memory — good for consistent lighting. Higher alpha (0.5+) reacts faster to sudden changes — useful if a vehicle enters a tunnel mid-track.

Strategy C — Quality-Gated Vote

The most accurate strategy. Each vote is weighted by both the SGIE's softmax confidence and the frame quality score from your best-frame-selection scorer. Frames with poor geometry, edge truncation, or low tracker confidence contribute proportionally less.

Python
CONF_GATE  = 0.50   # min SGIE confidence to participate in vote
MIN_VOTES  = 5     # require at least N quality frames

def quality_gated_color(color_history, quality_scores):
    """
    color_history:  [(label, sgie_conf), ...]
    quality_scores: [float, ...]  — from frame_quality_score()
                    must be same length as color_history
    """
    pairs = [
        (lbl, sgie_conf, q_score)
        for (lbl, sgie_conf), q_score
        in zip(color_history, quality_scores)
        if sgie_conf > CONF_GATE
    ]

    # Fall back to unfiltered if not enough quality frames
    if len(pairs) < MIN_VOTES:
        pairs = [
            (lbl, c, 1.0)
            for lbl, c in color_history
        ]

    weight_map = defaultdict(float)
    for lbl, sgie_conf, q_score in pairs:
        # Weight = product of both quality signals
        weight_map[lbl] += sgie_conf * q_score

    if not weight_map:
        return None, 0.0

    stable_label = max(weight_map, key=weight_map.get)
    total        = sum(weight_map.values())
    stability    = weight_map[stable_label] / total

    return stable_label, stability

Full Probe Integration

Extend the existing best-frame buffer probe. Color label and confidence are appended alongside the frame score and crop bytes, then consumed at flush time.

Python · buffer_probe (extended)
def buffer_probe(pad, info, u_data):
    gst_buffer = info.get_buffer()
    batch_meta = pyds.gst_buffer_get_nvds_batch_meta(hash(gst_buffer))

    l_frame = batch_meta.frame_meta_list
    while l_frame:
        frame_meta = pyds.NvDsFrameMeta.cast(l_frame.data)
        src_id     = frame_meta.source_id

        l_obj = frame_meta.obj_meta_list
        while l_obj:
            obj_meta = pyds.NvDsObjectMeta.cast(l_obj.data)
            key      = (src_id, obj_meta.object_id)
            rect     = obj_meta.rect_params

            score               = frame_quality_score(obj_meta.confidence, rect, frame_meta)
            crop                = extract_crop(gst_buffer, frame_meta, rect)
            color_label, c_conf = get_color_prediction(obj_meta)  # SGIE result

            object_frame_buffer[key].append({
                "score":       score,
                "frame_num":   frame_meta.frame_num,
                "bbox":        (rect.left, rect.top, rect.width, rect.height),
                "crop":        cv2.imencode('.jpg', crop)[1].tobytes(),
                "color_label": color_label,   # None if SGIE not available
                "color_conf":  c_conf,
            })

            last_seen[key] = frame_meta.frame_num
            l_obj = l_obj.next
        l_frame = l_frame.next

    _flush_terminated_tracks(batch_meta)
    return Gst.PadProbeReturn.OK


def flush_and_select(key):
    src_id, oid = key
    frames = object_frame_buffer.get(key, [])
    if not frames:
        return

    # Best frames / crops (as before)
    sorted_f   = sorted(frames, key=lambda x: x["score"], reverse=True)
    top_frames = sorted_f[:3]
    top_crops  = sorted_f[:2]

    # Stable color via quality-gated vote
    color_entries = [
        (e["color_label"], e["color_conf"])
        for e in frames
        if e["color_label"] is not None
    ]
    quality_scores = [
        e["score"]
        for e in frames
        if e["color_label"] is not None
    ]
    stable_color, stability = quality_gated_color(color_entries, quality_scores)

    result = {
        "object_id":    oid,
        "source_id":    src_id,
        "stable_color": stable_color,
        "stability":    round(stability, 3),
        "n_color_votes": len(color_entries),
        "top_frames":   top_frames,
        "top_crops":    top_crops,
    }
    # → push to Kafka / MariaDB / Milvus

    del object_frame_buffer[key]
    last_seen.pop(key, None)

Handling Ambiguous Cases

Stability ScoreInterpretationAction
> 0.80
Clean, unambiguous color. Near-unanimous vote. Accept. Write to DB with color_confidence=HIGH.
0.50 – 0.80
Likely correct but some ambiguity. Silver/gray split common here. Accept with color_confidence=MED flag. Exclude from strict ReID filter.
< 0.50
Highly ambiguous. Votes spread across 3+ labels. Mark as unknown. Do not use for ReID color filtering.
n_votes < 5 Too few observations — object too brief or too small. Defer. Do not commit a color label.
Hardest Pairs silver ↔ gray and white ↔ silver are the two most confused pairs — both are specular and highly lighting-sensitive. Consider merging them into a single achromatic class if your downstream application doesn't need the distinction. Accuracy on the merged class is typically 15–20 percentage points higher.
