Overview

01 The Core Problem

Your pipeline emits per-frame detections with bounding boxes, class labels, and confidence scores. The goal is to collect all detections belonging to the same physical object and select the highest-quality frames and crops from that collection.

Top-3 Frames

Best 3 full-frame snapshots per object — useful for review, audit trails, and multi-frame majority voting.

Top-2 Crops

Best 2 tight bbox crops per object — primary output for ReID gallery ingestion and classifier fine-tuning.

Three sub-problems must be solved in sequence:

  • Association — which detections belong to the same object across frames?
  • Scoring — which of those frames are highest quality?
  • Flushing — when do we know a track is complete and ready to process?

Quality Metric

03 Frame Quality Score

Raw detection confidence is a weak quality proxy on its own. A composite score across multiple signals is significantly more reliable for selecting clean, well-cropped frames.

Composite Score Formula
score = conf × 0.5 // PGIE confidence (0–1)
+ size_score × 0.4 // bbox area / frame area, capped at 1.0
+ aspect_ok × 0.1 // 0.4 < w/h < 2.5 → 1, else 0
Python
def frame_quality_score(conf, rect, frame_meta):
    w = rect.width
    h = rect.height
    area = w * h

    # Penalize extreme aspect ratios (occluded / edge-clipped vehicles)
    aspect_ok = float(0.4 < (w / max(h, 1)) < 2.5)

    # PGIE confidence
    conf_score = conf

    # Normalize bbox area to frame area
    frame_area = frame_meta.source_frame_width * frame_meta.source_frame_height
    size_score = min(area / max(frame_area, 1), 1.0)

    # Optional: penalize objects near frame edges (likely truncated)
    cx = rect.left + w / 2
    cy = rect.top  + h / 2
    margin_x = cx / frame_meta.source_frame_width
    margin_y = cy / frame_meta.source_frame_height
    edge_ok = float(0.05 < margin_x < 0.95 and
                    0.05 < margin_y < 0.95)

    score = (conf_score   * 0.45 +
             size_score   * 0.35 +
             aspect_ok    * 0.10 +
             edge_ok      * 0.10)
    return score

Occlusion proxy Overlap with other bboxes in frame Low overlap = unoccluded object
Laplacian variance cv2.Laplacian on crop → variance Direct sharpness estimate
GStreamer Integration

04 Accumulation Probe

Attach a pad probe on the sink pad of nvdsosd. At this point PGIE and NvDCF metadata are fully populated but OSD has not yet burned text into the frame — bbox geometry is still accurate for cropping.

Critical: Pixel Data Lifetime
NvBufSurface pointers are only valid inside the probe callback. You must copy crop pixel data before returning Gst.PadProbeReturn.OK. Storing the frame metadata pointer and reading pixels later is a use-after-free bug.
Python · buffer_probe
import pyds
import numpy as np
import cv2
from collections import defaultdict

object_frame_buffer = defaultdict(list)  # (src_id, oid) → [entries]
last_seen            = {}                # (src_id, oid) → frame_num
BUFFER_CAP           = 50              # max frames stored per object

def extract_crop(gst_buffer, frame_meta, rect):
    n_frame = pyds.get_nvds_buf_surface(
        hash(gst_buffer), frame_meta.batch_id
    )
    frame_bgr = np.array(n_frame, copy=True)
    fh, fw    = frame_bgr.shape[:2]
    x1 = max(0,  int(rect.left))
    y1 = max(0,  int(rect.top))
    x2 = min(fw, int(rect.left + rect.width))
    y2 = min(fh, int(rect.top  + rect.height))
    return frame_bgr[y1:y2, x1:x2]

def buffer_probe(pad, info, u_data):
    gst_buffer = info.get_buffer()
    batch_meta = pyds.gst_buffer_get_nvds_batch_meta(hash(gst_buffer))

    # ── accumulate live detections ──
    l_frame = batch_meta.frame_meta_list
    while l_frame:
        frame_meta = pyds.NvDsFrameMeta.cast(l_frame.data)
        src_id     = frame_meta.source_id
        frame_num  = frame_meta.frame_num

        l_obj = frame_meta.obj_meta_list
        while l_obj:
            obj_meta = pyds.NvDsObjectMeta.cast(l_obj.data)
            key      = (src_id, obj_meta.object_id)
            rect     = obj_meta.rect_params

            score = frame_quality_score(
                obj_meta.confidence, rect, frame_meta
            )
            crop  = extract_crop(gst_buffer, frame_meta, rect)

            buf = object_frame_buffer[key]
            buf.append({
                "score":     score,
                "frame_num": frame_num,
                "bbox":      (rect.left, rect.top,
                               rect.width, rect.height),
                "crop":      cv2.imencode('.jpg', crop)[1].tobytes(),
            })

            # Cap buffer — evict lowest-score entry
            if len(buf) > BUFFER_CAP:
                buf.sort(key=lambda x: x["score"], reverse=True)
                buf.pop()  # remove worst

            last_seen[key] = frame_num
            l_obj = l_obj.next
        l_frame = l_frame.next

    # ── past frame flush (see Section 05) ──
    _flush_terminated_tracks(batch_meta, gst_buffer)

    return Gst.PadProbeReturn.OK

# Attach to nvdsosd sink pad — after NvDCF metadata is populated
osd_sink_pad = nvdsosd.get_static_pad("sink")

Track Termination Signal

05 NvDCF Past Frame Metadata

NvDCF does not fire a GStreamer event when a track ends. Instead, it writes terminated track data into a NvDsPastFrameObjBatch structure attached to the same GstBuffer — a parallel metadata path alongside the normal frame meta list.

Required Tracker Config

config_tracker_NvDCF.yml
NvDCF:
  # ... other params ...
  pastFrame: 1               # must be enabled
  maxShadowTrackingAge: 30    # frames in shadow before termination
  outputTerminatedTracks: 1  # surfaces terminated data in batch
Config Requirement
Without pastFrame: 1, the past_frame_obj_batch pointer will be null and no termination signals will be surfaced. The periodic timeout fallback (Section 06) will still work, but you lose the authoritative flush signal.

Past Frame Metadata Hierarchy

NvDsPastFrameObjBatch — one per GstBuffer batch └── NvDsPastFrameObjStream streamID, numList — one per source └── NvDsPastFrameObjList uniqueId, numObj — one per terminated track └── NvDsPastFrameObj frameNum, tBbox, confidence, age — one entry per frame
Pixel Data Note
Past frame metadata provides bbox geometry and confidence history — but not pixel data. The GPU frame buffers those frames lived in are long gone. This is why crops must be extracted live in the current-frame probe and stored in the accumulator buffer.
Python · past frame traversal
def _flush_terminated_tracks(batch_meta, gst_buffer):
    past_batch = pyds.nvds_get_pastframe_obj_batch(
        batch_meta.past_frame_obj_batch
    )
    if past_batch is None:
        return

    l_stream = past_batch.list
    while l_stream:
        stream = pyds.NvDsPastFrameObjStream.cast(l_stream.data)
        src_id = stream.streamID

        l_track = stream.list
        while l_track:
            track = pyds.NvDsPastFrameObjList.cast(l_track.data)
            oid   = track.uniqueId
            key   = (src_id, oid)

            if key in object_frame_buffer:
                flush_and_select(key)

            l_track = l_track.next
        l_stream = l_stream.next
Selection Logic

06 Flush & Select Best-N

Primary: Past Frame Flush

Triggered by NvDCF termination signal — authoritative and immediate.

Safety Net: Periodic Timeout Flush

Objects still visible when the video ends never generate a termination signal. Run a timeout sweep every N frames as a safety net:

Python
TIMEOUT_FRAMES = 60  # 2 seconds at 30fps

def flush_stale_tracks(current_frame_num):
    stale = [
        k for k, f in last_seen.items()
        if current_frame_num - f > TIMEOUT_FRAMES
    ]
    for key in stale:
        flush_and_select(key)
        del last_seen[key]

The flush_and_select Function

Python
import os

TOP_FRAMES = 3
TOP_CROPS  = 2

def flush_and_select(key):
    src_id, oid = key
    frames = object_frame_buffer.get(key, [])

    if not frames:
        object_frame_buffer.pop(key, None)
        last_seen.pop(key, None)
        return

    # Sort descending by quality score
    sorted_frames = sorted(frames, key=lambda x: x["score"], reverse=True)

    top_frames = sorted_frames[:TOP_FRAMES]
    top_crops  = sorted_frames[:TOP_CROPS]

    os.makedirs(f"out/frames/src{src_id}", exist_ok=True)
    os.makedirs(f"out/crops/src{src_id}",  exist_ok=True)

    for rank, entry in enumerate(top_frames, 1):
        out_path = (
            f"out/frames/src{src_id}/"
            f"obj{oid}_rank{rank}_f{entry['frame_num']}.jpg"
        )
        with open(out_path, "wb") as fh:
            fh.write(entry["crop"])  # stored as JPEG bytes

    for rank, entry in enumerate(top_crops, 1):
        out_path = (
            f"out/crops/src{src_id}/"
            f"obj{oid}_crop{rank}.jpg"
        )
        with open(out_path, "wb") as fh:
            fh.write(entry["crop"])

    # Clean up
    del object_frame_buffer[key]
    last_seen.pop(key, None)
Memory Tip
Store crops as cv2.imencode('.jpg', crop)[1].tobytes() — compressed JPEG bytes use ~10–40× less memory than raw numpy arrays. At 50 frames/object with 200×100px crops, raw = ~3MB per object; JPEG bytes ≈ 100–300KB.
Architecture

07 Summary Flow

Frame N arrives in buffer_probe
  │
  ├─ for each detection in frame
  │    ├─ key = (source_id, object_id)
  │    ├─ score = composite_score(conf, bbox, frame_meta)
  │    ├─ crop  = extract_crop(surface, bbox)        ← must happen NOW
  │    ├─ append {score, frame_num, bbox, crop_bytes}
  │    └─ if len(buffer[key]) > CAP: evict worst entry
  │
  └─ check past_frame_obj_batch
       └─ for each terminated track_id
            └─ flush_and_select((src_id, track_id))

Every 60 frames (safety net):
  └─ flush_stale_tracks(current_frame_num)

flush_and_select(key):
  ├─ sort buffer[key] by score DESC
  ├─ write top[:3] to out/frames/  ← top-3 frames
  ├─ write top[:2] to out/crops/   ← top-2 crops
  └─ del buffer[key]
Production Considerations

08 Practical Notes

ConcernRecommendation
Memory cap Hard cap at 50 frames/object; evict lowest-score entry on overflow. Prevents unbounded growth for long-duration tracks.
Crop storage format Store as JPEG bytes in-memory via cv2.imencode. 10–40× smaller than raw numpy arrays.
Multi-stream keying Always use (source_id, object_id) tuple as buffer key — same object_id values appear across streams.
maxShadowTrackingAge Set to expected occlusion duration in your scene. 30–60 frames typical for vehicle tracking at 30fps.
Downstream integration In flush_and_select, push top crops directly to a Celery task or Milvus insert rather than writing to disk — avoids I/O bottleneck at high vehicle density.
End-of-stream flush Attach an EOS pad probe or listen to bus.MESSAGE_EOS to flush all remaining buffers when the video ends.
Thread safety GStreamer probes run on the streaming thread. If you push crops to Celery or another thread, use a queue.Queue as the handoff — do not call Celery directly from the probe.
GPU Memory Abstraction

09 NvBufSurface — The GPU Frame Buffer

A NvBufSurface is DeepStream's core abstraction for GPU-accessible video frame memory. Rather than passing raw CPU buffers between elements (which would require expensive CPU↔GPU copies at every stage), every plugin in the chain operates directly on GPU memory — zero copies, full throughput.

Core Idea
Think of a surface as a tile sheet of frames living on the GPU. PGIE, SGIEs, NvDCF, and OSD all read and write to this same tile sheet without ever touching CPU memory. Your Python probes operate on the metadata sidecar (NvDsBatchMeta) that rides alongside the surface — not on pixels — unless you explicitly map the surface.

Batching and nvstreammux

nvstreammux is what creates the batched surface. It collects frames from multiple RTSP streams and packs them into a single NvBufSurface batch, enabling TensorRT to run one inference call on the whole batch rather than N separate calls — the primary throughput multiplier for multi-stream pipelines.

Accessing Surface Pixels in a Probe

Use pyds.get_nvds_buf_surface() to map the surface into a numpy array. This is valid only inside the probe callback — the buffer is returned to the pool the moment you return Gst.PadProbeReturn.OK.

Python · surface access in probe
def buffer_probe(pad, info, u_data):
    gst_buffer = info.get_buffer()
    batch_meta = pyds.gst_buffer_get_nvds_batch_meta(hash(gst_buffer))

    l_frame = batch_meta.frame_meta_list
    while l_frame:
        frame_meta = pyds.NvDsFrameMeta.cast(l_frame.data)

        # Map GPU surface → numpy array (RGBA or NV12 depending on pipeline)
        n_frame = pyds.get_nvds_buf_surface(
            hash(gst_buffer),
            frame_meta.batch_id   # index into surfaceList[]
        )

        # Copy immediately — pointer is invalid after probe returns
        frame_copy = np.array(n_frame, copy=True)
        frame_bgr  = cv2.cvtColor(frame_copy, cv2.COLOR_RGBA2BGR)

        # ... crop objects from frame_bgr ...

        l_frame = l_frame.next

    return Gst.PadProbeReturn.OK
Lifetime Warning
NvBufSurface pointers returned by get_nvds_buf_surface() are only valid inside the probe. Storing the pointer and reading pixels later is a use-after-free bug. Always np.array(..., copy=True) before exiting the probe.

Why This Matters for Best-Frame Selection

The entire crop extraction pipeline in Section 04 is built on this constraint. You must extract and compress the crop (cv2.imencode) within the probe, then store the resulting JPEG bytes — not the raw numpy array and certainly not a pointer to the GPU buffer.

Tracker History API

10 NvDsPastFrameObjBatch — Deep Dive

NvDsPastFrameObjBatch is the structure the NvDCF tracker populates when past-frame data reporting is enabled. It exposes detection history that was buffered internally by the tracker during uncertainty windows — detections that were held back from the normal NvDsObjectMeta path while the tracker was still resolving track identity.

Why It Exists

NvDCF uses an internal buffer to smooth detections, handle occlusion, and maintain track continuity. During this buffering period, a frame may pass through the pipeline without a confirmed detection surfaced in NvDsObjectMeta — but the tracker still held a candidate internally. NvDsPastFrameObjBatch flushes that history out when a track terminates or when the shadow tracking window expires.

Use Case: Trajectory Reconstruction

Forensic reconstruction of a track's full detection timeline, including frames where the object was present but the tracker hadn't yet confirmed the detection.

Use Case: LP / Attribute Gap Fill

Recover bbox history for frames where LP reads or color classifications were skipped during the tracker's uncertainty window — retry attribute extraction against those frames retroactively.

Full Structure Hierarchy

NvDsPastFrameObjBatch — one per GstBuffer (entire batch) └── list[] — array of per-stream containers └── NvDsPastFrameObjStream streamID, surfaceStreamID, numFilled — one per source_id └── NvDsPastFrameObjList uniqueId, classId, numObj — one per tracked object └── NvDsPastFrameObj frameNum, tBbox, confidence, age — one entry per buffered frame

Field Reference

FieldTypeNotes
streamID uint32 Matches frame_meta.source_id — use as composite key prefix
numFilled uint32 Objects with history in this stream. Always loop to numFilled, not numAllocated
uniqueId uint64 Same object_id as in NvDsObjectMeta — direct correlation key
numObj uint32 Number of past frame entries in objList[] for this track
frameNum uint64 Matches NvDsFrameMeta.frame_num — chronological order, oldest first
tBbox NvDsComp_BboxInfo Tracker bbox (.m_unClippedRect → left, top, width, height)
confidence float Tracker confidence for this buffered detection
age uint32 Frames elapsed since track was first created

Full Python Traversal Pattern

Python · NvDsPastFrameObjBatch traversal
import pyds

def tracker_src_probe(pad, info, u_data):
    gst_buffer = info.get_buffer()
    batch_meta = pyds.gst_buffer_get_nvds_batch_meta(hash(gst_buffer))

    # Past frame data arrives as user meta on the batch
    l_usr = batch_meta.batch_user_meta_list
    while l_usr is not None:
        try:
            usr_meta = pyds.NvDsUserMeta.cast(l_usr.data)
        except StopIteration:
            break

        if (usr_meta.base_meta.meta_type ==
                pyds.NvDsMetaType.NVDS_TRACKER_PAST_FRAME_META):

            pf_batch = pyds.NvDsPastFrameObjBatch.cast(
                usr_meta.user_meta_data
            )

            # Iterate streams
            for s_idx in range(pf_batch.numAllocated):
                pf_stream = pf_batch.list[s_idx]
                src_id    = pf_stream.streamID

                # Iterate tracked objects with history
                for o_idx in range(pf_stream.numFilled):
                    pf_obj_list = pf_stream.list[o_idx]
                    track_id    = pf_obj_list.uniqueId
                    key         = (src_id, track_id)

                    # Iterate buffered past frames for this track
                    for f_idx in range(pf_obj_list.numObj):
                        pf = pf_obj_list.objList[f_idx]
                        bbox = pf.tBbox.m_unClippedRect
                        print(
                            f"Track {track_id} | "
                            f"frame {pf.frameNum} | "
                            f"conf {pf.confidence:.3f} | "
                            f"age {pf.age} | "
                            f"bbox ({bbox.left:.0f},{bbox.top:.0f},"
                            f"{bbox.width:.0f},{bbox.height:.0f})"
                        )

                    # Flush accumulator if key exists
                    if key in object_frame_buffer:
                        flush_and_select(key)

        try:
            l_usr = l_usr.next
        except StopIteration:
            break

    return Gst.PadProbeReturn.OK

Relationship to Active Metadata

Mutual Exclusivity
For any given frame, a detection appears in either the normal NvDsObjectMeta on NvDsFrameMeta or in past frame data — never both. Past frame data represents detections that were held back during the tracker's internal buffering window. uniqueId in NvDsPastFrameObjList is the same object_id you see in active object meta, enabling clean cross-correlation.

Gap-Filling Track History Pattern

Combine past frame data with your live accumulator to reconstruct a lossless per-track detection timeline:

Python · merge past frames into accumulator
def merge_past_frames_into_buffer(pf_batch, track_history):
    for s_idx in range(pf_batch.numAllocated):
        stream = pf_batch.list[s_idx]
        for o_idx in range(stream.numFilled):
            obj = stream.list[o_idx]
            tid = obj.uniqueId
            key = (stream.streamID, tid)

            if key not in track_history:
                track_history[key] = []

            for f_idx in range(obj.numObj):
                pf = obj.objList[f_idx]
                track_history[key].append({
                    "frame":  pf.frameNum,
                    "conf":   pf.confidence,
                    "age":    pf.age,
                    "bbox":   pf.tBbox.m_unClippedRect,
                    "source": "past_frame",  # distinguish from live detections
                })
Integration Tip
Call merge_past_frames_into_buffer() from your main buffer probe alongside the live accumulation loop. Past frame entries carry bbox geometry and confidence but no pixel data — the GPU surface for those frames is long gone. Use them to score and rank, but crop pixels must come from live frames captured in the same probe pass.

Track Lifecycle

12 Detecting Terminated Tracks

NvDsPastFrameObjBatch has no termination flag — it only tells you a track is in shadow phase, not that it has ended. Termination must be inferred by cross-referencing both metadata channels across consecutive batches.

Common Misconception
A track appearing in NvDsPastFrameObjBatch does not mean it has terminated. It means the tracker is holding back detections during its shadow phase. The track is still alive — it only terminates when it vanishes from both channels simultaneously.

The Two Channels

NvDsObjectMeta (active)

Track is confirmed by the detector this frame. Appears in frame_meta.obj_meta_list. Object is currently visible.

NvDsPastFrameObjBatch (shadow)

Track is in shadow phase — detector lost it but tracker is still predicting. No longer in active meta. Still alive, awaiting re-detection.

Track Lifecycle vs Metadata Presence

Batch N-1:   track 42 in NvDsObjectMeta        → known_ids has (0, 42)
Batch N:     track 42 in NvDsPastFrameObjBatch  → still_alive has (0, 42) — NOT terminated
Batch N+1:   track 42 absent from both          → terminated = {(0, 42)}  ✓

Implementation

Python · collect active ids
def get_active_ids(batch_meta):
    """IDs currently confirmed by detector this batch."""
    active = set()
    l_frame = batch_meta.frame_meta_list
    while l_frame:
        frame_meta = pyds.NvDsFrameMeta.cast(l_frame.data)
        l_obj = frame_meta.obj_meta_list
        while l_obj:
            obj = pyds.NvDsObjectMeta.cast(l_obj.data)
            active.add((frame_meta.source_id, obj.object_id))
            l_obj = l_obj.next
        l_frame = l_frame.next
    return active
Python · collect shadow ids
def get_shadow_ids(batch_meta):
    """IDs currently in shadow phase (in NvDsPastFrameObjBatch)."""
    shadow = set()
    l_usr = batch_meta.batch_user_meta_list
    while l_usr:
        try:
            usr_meta = pyds.NvDsUserMeta.cast(l_usr.data)
        except StopIteration:
            break
        if (usr_meta.base_meta.meta_type ==
                pyds.NvDsMetaType.NVDS_TRACKER_PAST_FRAME_META):
            pf_batch = pyds.NvDsPastFrameObjBatch.cast(usr_meta.user_meta_data)
            for s in range(pf_batch.numAllocated):
                stream = pf_batch.list[s]
                for o in range(stream.numFilled):
                    obj = stream.list[o]
                    shadow.add((stream.streamID, obj.uniqueId))
        try:
            l_usr = l_usr.next
        except StopIteration:
            break
    return shadow
Python · infer terminations
def find_terminated(batch_meta, known_ids):
    """
    known_ids : set of (src_id, object_id) seen in any previous batch.
    Returns   : set of IDs that just terminated this batch.
    Mutates known_ids in-place for the next batch.
    """
    active      = get_active_ids(batch_meta)
    shadow      = get_shadow_ids(batch_meta)
    still_alive = active | shadow

    terminated  = known_ids - still_alive  # was known, now gone from both

    known_ids.clear()
    known_ids.update(still_alive)

    return terminated

    gst_buffer = info.get_buffer()
    batch_meta  = pyds.gst_buffer_get_nvds_batch_meta(hash(gst_buffer))

    terminated = find_terminated(batch_meta, known_ids)

    for (src_id, oid) in terminated:
        key = (src_id, oid)
        if key in object_frame_buffer:
            flush_and_select(key)  # now authoritative — track is truly done

    return Gst.PadProbeReturn.OK

Edge Case: Long Shadow Windows

If maxShadowTrackingAge is large (e.g. 60 frames) a vehicle that exits frame cleanly will sit in shadow phase for up to 2 seconds before termination fires. The timeout flush from Section 06 remains the right safety net — run both in parallel, they complement each other:

MechanismWhen it firesRole
Dual-channel termination Exactly when track vanishes from both channels Authoritative flush — use as primary signal
Timeout flush (Section 06) After N frames absent from last_seen Safety net for EOS, long shadow windows, edge cases
Why Not Use outputTerminatedTracks?
The tracker config flag outputTerminatedTracks: 1 surfaces a termination event, but it requires additional event bus handling and is less portable across DeepStream versions. The dual-channel set-difference approach above works purely from probe metadata with no extra config dependencies.
