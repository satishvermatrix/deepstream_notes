01 Model & Engine Model Loading

Parameters controlling how nvinfer locates, loads, and caches the TensorRT engine. Engine files are built once and cached — subsequent runs skip the build step.

Parameter Type Default Values Description Applies To
model-engine-file★ string required path/to/model.engine Path to the serialized TensorRT engine. If the file doesn't exist, nvinfer builds it from the model source and saves it here. Always pre-build on the target GPU for reproducibility. Both
onnx-file string — path/to/model.onnx Path to ONNX model. Used to build the TRT engine if model-engine-file doesn't exist. One of onnx-file, tlt-encoded-model, or model-file must be provided if no engine exists. Both
tlt-encoded-model string — path/to/model.etlt Path to TAO/TLT encrypted model (.etlt). Requires tlt-model-key for decryption. Used for NVIDIA TAO Toolkit models. Both
tlt-model-key string — string Encryption key for decrypting tlt-encoded-model files. Required when using TAO Toolkit encrypted models. Both
model-file string — .caffemodel Path to Caffe model weights file. Caffe support is legacy — ONNX is preferred for all new models. Both
proto-file string — .prototxt Path to Caffe network prototxt. Required alongside model-file for Caffe models. Both
labelfile-path string — labels.txt Path to text file with one class label per line. Line index maps to class ID. Used by nvinfer to populate result_label in NvDsLabelInfo. Both
gpu-id uint 0 01N GPU device index to run inference on. All GIEs in a pipeline must share the same gpu-id unless using multi-GPU explicit mode. Both
gie-unique-id★ uint required 123… Unique identifier for this GIE instance. Used to tag NvDsClassifierMeta.unique_component_id and NvDsInferTensorMeta.unique_id. PGIE typically uses 1. Each SGIE must have a distinct ID. Both
02 Input & Preprocessing Preprocessing

Controls how nvinfer normalizes pixel values before feeding them to the network. Must match exactly what was used during model training. Normalization mismatch silently degrades accuracy.

Normalization Mismatch Warning These values must exactly replicate your training preprocessing. A wrong net-scale-factor or offsets silently reduces accuracy by 15–30%. Formula: output = net-scale-factor × (pixel_value − offset)
Parameter Type Default Values Description Applies To
net-scale-factor★ float 1.0 1/255 = 0.00392 Multiplier applied to each pixel value after subtracting offsets. For [0,1] normalized nets: 0.00392156... (1/255). For ImageNet-normalized: 1/(255×std). Must match training. Both
offsets★ string 0 R;G;B Per-channel mean values subtracted from each pixel before scaling. Semicolon-separated R;G;B. ImageNet: 123.675;116.280;103.530 (= mean×255). Zero for simple /255 normalization. Both
infer-dims string from model C;H;W Override input tensor dimensions as Channel;Height;Width. Usually inferred from the ONNX model. Override only if model has dynamic input shapes. Both
network-input-order uint 0 0=NCHW1=NHWC Memory layout of the input tensor. 0=NCHW (PyTorch default), 1=NHWC (TensorFlow default). Most TensorRT models expect NCHW. Both
color-format uint 0 0=RGB1=BGR2=GRAY Expected input color format. DeepStream surfaces are NV12/RGBA — nvinfer converts automatically. Set to match your model's training color space. Most PyTorch/TF models expect RGB. Both
maintain-aspect-ratio bool 0 01 When 1, nvinfer letterboxes the input to preserve aspect ratio before resizing to infer-dims. Avoids object distortion. Requires symmetric-padding or custom preprocessing to be effective. Both
symmetric-padding bool 0 01 When 1 (with maintain-aspect-ratio=1), pads equally on both sides. When 0, pads on right/bottom only. Symmetric mode is required for accurate coordinate mapping post-resize. Both
input-object-min-width★ uint 0 pixels SGIE only. Objects with bbox width smaller than this value are skipped — no inference is run. Critical for quality gating: skip tiny/distant objects where the classifier would get noisy crops. SGIE
input-object-min-height uint 0 pixels SGIE only. Objects with bbox height smaller than this value are skipped. Use alongside input-object-min-width to define minimum crop quality gate. SGIE
input-object-max-width uint 0 (no limit) pixels SGIE only. Objects wider than this are skipped. Rarely used — useful for filtering out full-frame detections or very large overlapping boxes. SGIE
input-object-max-height uint 0 (no limit) pixels SGIE only. Objects taller than this are skipped. SGIE
interval uint 0 0=every frameN=every N+1 Number of consecutive frames to skip between inference. 0 = run on every frame. 1 = run every 2nd frame. Useful for reducing SGIE compute cost on high-fps streams. Both
input-tensor-from-meta bool 0 01 When 1, reads the input tensor from upstream NvDsUserMeta (e.g. from nvdspreprocess plugin) instead of performing internal preprocessing. Required when using nvdspreprocess for custom batching. Both
03 Network Type & Process Mode Network

The most fundamental configuration choices. network-type determines how nvinfer interprets the output tensor. process-mode determines whether inference runs on the full frame (PGIE) or on cropped objects (SGIE).

Parameter Type Default Values Description Applies To
network-type★ uint 0 0=detector 1=classifier 2=segmentation 3=instance seg 100=other Determines output interpretation. 0=detector: decodes bbox + NMS → NvDsObjectMeta. 1=classifier: argmax → NvDsClassifierMeta/LabelInfo. 2=segmentation: pixel-wise class map. 3=instance segmentation: mask per object. 100=other: raw tensor only, no built-in decode. Both
process-mode★ uint 1 1=primary2=secondary 1=primary (PGIE): runs on full frame. Creates NvDsObjectMeta entries. 2=secondary (SGIE): crops each object bbox from the frame surface and runs inference per crop. Requires operate-on-gie-id to identify which objects to process. Both
num-detected-classes uint required integer ≥ 1 Number of output classes. For detectors: total object classes including background if applicable. For classifiers: total label count. Must match model output exactly. Both
classifier-threshold float 0.5 [0.0, 1.0] Minimum softmax probability for the classifier to report a result via NvDsLabelInfo. Results below this threshold are suppressed. Lower values = more results but potentially noisy. Both
threshold float 0.3 [0.0, 1.0] Detector object confidence threshold. Detections below this confidence are discarded before NMS. Applies to network-type=0 only. Per-class thresholds can be set via per-class-attrs-group. PGIE
nms-iou-threshold float 0.5 [0.0, 1.0] IoU threshold for NMS post-processing. Boxes with IoU above this value are suppressed. Lower = fewer overlapping boxes. Typical: 0.3–0.6. Applies to detector models only. PGIE
pre-cluster-threshold float 0.3 [0.0, 1.0] Per-class confidence threshold applied before clustering/NMS. Filters raw detections before they enter the NMS pipeline. Set per-class via [class-attrs-all] or per-class groups. PGIE
cluster-mode uint 3 0=OpenCV groupRect 1=DBSCAN 2=NMS 3=NMS+groupRects 4=none Post-processing clustering algorithm for detector output. Mode 2 (NMS) is standard for YOLO models. Mode 4 (none) is used when the model includes built-in NMS (e.g. EfficientDet). PGIE
topk int -1 -1=no limitN=max N Maximum number of objects to output per frame after NMS. -1 = unlimited. Setting a cap limits downstream metadata volume. Typical: 200–2000 for dense scenes. PGIE
classifier-async-mode★ bool 0 0=sync1=async 0=sync: SGIE results appear on the same frame as the detection that triggered them. Simpler to reason about in probes. 1=async: Results appear on a later frame (lag of 1 batch). Improves throughput but requires lag handling in probe code. Applies to classifiers only. SGIE
segmentation-threshold float 0.5 [0.0, 1.0] For network-type=2 (segmentation). Pixel probability threshold — pixels below this value are assigned the background class. Applies only to segmentation models. Both
04 Output & Tensor Control Output

Controls what metadata nvinfer attaches to the buffer — whether to surface raw tensors (for custom post-processing) or only decoded labels/bboxes.

Parameter Type Default Values Description Applies To
output-tensor-meta★ bool 0 0=label mode1=tensor mode 0: nvinfer runs argmax internally and attaches NvDsClassifierMeta (label + top-1 prob only). 1: nvinfer attaches NvDsInferTensorMeta containing the raw output tensor buffer. You get the full softmax vector. Mutually exclusive — enabling 1 disables NvDsClassifierMeta attachment. Both
output-instance-mask bool 0 01 For network-type=3 (instance segmentation). When 1, attaches per-object mask tensors to NvDsObjectMeta. Requires significant memory bandwidth per object. PGIE
output-blob-names string all outputs name1;name2 Semicolon-separated names of output layer blobs to expose when output-tensor-meta=1. Use to select specific output heads from a multi-output model. Leave empty to expose all. Both
secondary-reinfer-interval uint 0 0=every frameN=every Nth SGIE only. Re-run inference on an object every N frames even if the result is cached. 0 = re-infer every frame. Higher values reduce GPU load for slowly-changing attributes like color. SGIE
05 SGIE Chaining SGIE Only

These parameters only apply to SGIEs (process-mode=2). They define which upstream GIE's objects this SGIE should process, which classes to run on, and how to filter by bbox size.

Parameter Type Default Values Description Applies To
operate-on-gie-id★ uint required 12N gie-unique-id of the upstream GIE whose objects this SGIE should process. For a vehicle color classifier: set to the PGIE's gie-unique-id (usually 1). For LP char classifier chained after LP detector: set to LP detector's id. SGIE
operate-on-class-ids★ string all classes 00;1;2 Semicolon-separated list of class IDs from the upstream GIE to run inference on. Empty = all classes. Set to "0" to only classify vehicles (class 0) and skip pedestrians/bikes. Reduces SGIE compute significantly. SGIE
06 Performance & TensorRT Performance

TensorRT engine build options, precision, batch size, and workspace memory. Engine build options are used only during the initial build phase — they don't affect a pre-built cached engine.

Parameter Type Default Values Description Applies To
batch-size★ uint 1 1–N Maximum number of frames (PGIE) or object crops (SGIE) processed in a single TRT inference call. Higher batch sizes improve GPU utilization but increase latency. For SGIE with many objects per frame, set to 16–64. Both
workspace-size uint (MB) 1024 MB Maximum GPU memory (MB) allocated as TensorRT workspace during engine build. Larger workspace allows TRT to find faster kernel implementations. 512–4096 MB typical. Does not affect inference memory. Both
enable-dla bool 0 01 Enable NVIDIA DLA (Deep Learning Accelerator) for Jetson AGX Orin. DLA is separate from GPU — runs independently. Frees GPU for other tasks. Only available on Jetson platforms with DLA hardware. Both
use-dla-core uint 0 01 Select which DLA core to use (0 or 1). Jetson AGX Orin has 2 DLA cores. Requires enable-dla=1. Both
int8-calib-file string — path/to/calib.bin Path to INT8 calibration cache file. When provided, TRT builds an INT8 engine. INT8 is ~2× faster than FP16 but requires calibration data. Use trtexec to generate the calibration file separately. Both
enable-int8 bool 0 01 Enable INT8 precision for TRT engine build. Requires int8-calib-file or a custom calibrator. Fastest precision mode but requires careful calibration to avoid accuracy degradation. Both
enable-fp16 bool 0 01 Enable FP16 precision for TRT engine build. ~2× faster than FP32 on Ampere/Ada GPUs with minimal accuracy loss. Recommended default for production deployment. L40S supports FP16. Both
force-implicit-batch-dim bool 0 01 Force TRT to use implicit batch dimension mode (legacy TRT API). 0 = explicit batch (modern, supports dynamic shapes). Leave at 0 for ONNX models exported with dynamic=True. Both
nvbuf-memory-type uint 0 0=default 1=cuda pinned 2=cuda device 3=surface array Memory type for NvBufSurface allocation. 0 = platform default. 1 = CUDA pinned (faster host-device transfers). 2 = CUDA device memory. 3 = CUDA surface arrays. Usually leave at 0 unless profiling memory bottlenecks. Both
async-mode bool 1 0=sync1=async Whether nvinfer performs inference asynchronously with a dedicated thread. 1 (default) maximizes pipeline throughput. 0 = synchronous (simpler debugging). Distinct from classifier-async-mode. Both
infer-raw-output-dir string — path/to/dir/ Debug: directory path to dump raw inference output tensors to disk as binary files. Each frame produces one file per output layer. Useful for debugging output-tensor-meta extraction issues. Both
07 Custom Parsers & Libraries Custom

For non-standard model output formats (e.g. custom YOLO variants, specialized detectors) nvinfer supports shared library plugins for custom bbox decoding and network post-processing.

Parameter Type Default Values Description Applies To
parse-bbox-func-name string — NvDsInferParseCustomYolo Name of the C function in custom-lib-path to call for bbox decoding. The function must match the NvDsInferParseCustomFunc signature. Required for YOLO and any model not using standard SSD/UFF output format. PGIE
custom-lib-path string — libcustom.so Path to a shared library (.so) containing custom parsing functions. Used together with parse-bbox-func-name, parse-classifier-func-name, or engine-create-func-name. Both
parse-classifier-func-name string — Name of the C function for custom classifier output decoding. Overrides nvinfer's built-in argmax. Use when your classifier output needs custom softmax, multi-label, or multi-head decode logic. Both
engine-create-func-name string — NvDsInferEngineCreateCustomFunc Name of a custom function (in custom-lib-path) to build the TRT engine. Used when standard ONNX/Caffe parsing is insufficient — e.g. for models requiring custom TRT plugins or layers. Both
network-mode uint 2 0=FP32 1=INT8 2=FP16 Shorthand precision setting for the TRT engine. 0=FP32, 1=INT8 (requires calib file), 2=FP16. Equivalent to enabling enable-fp16=1 or enable-int8=1. network-mode=2 is the standard production setting. Both
scaling-compute-hw uint 0 0=GPU1=VIC Hardware used for image scaling/preprocessing. 0=GPU (default, supports all formats). 1=VIC (Jetson video image composer, offloads GPU). VIC is only available on Jetson platforms. Both
scaling-filter uint 0 0=nearest 1=bilinear 2=bilinear+edge 3=bilinear+border 4=bilinear+constant 5=bicubic 6=bicubic+edge Interpolation algorithm used when resizing input to infer-dims. 0=nearest (fastest). 1=bilinear (balanced). 5=bicubic (best quality, slower). For color classification, bilinear (1) is recommended. Both
EX PGIE Example — Vehicle Detector Example
config_pgie_vehicle.txtPGIE · YOLOv8 Detector
[property] gie-unique-id = 1 # all SGIEs reference this process-mode = 1 # 1 = PGIE (full frame) network-type = 0 # 0 = detector # Model model-engine-file = yolov8s_vehicle.engine labelfile-path = vehicle_labels.txt gpu-id = 0 # Preprocessing — YOLO uses simple /255 normalization net-scale-factor = 0.0039215697906911373 # 1/255 offsets = 0;0;0 # no mean subtraction color-format = 0 # 0 = RGB infer-dims = 3;640;640 # C;H;W # Detection params num-detected-classes = 4 # vehicle, person, bike, truck cluster-mode = 2 # 2 = NMS (for YOLO) nms-iou-threshold = 0.45 threshold = 0.25 # Custom YOLO parser (required for YOLOv8) parse-bbox-func-name = NvDsInferParseCustomYoloV8 custom-lib-path = libnvdsinfer_custom_impl_Yolo.so # TensorRT network-mode = 2 # 2 = FP16 batch-size = 4 workspace-size = 2048 # Output output-tensor-meta = 0 # standard NvDsObjectMeta

config_sgie_color.txtSGIE · 15-class classifier · tensor mode
[property] gie-unique-id = 2 process-mode = 2 # 2 = SGIE (object crops) network-type = 1 # 1 = classifier # SGIE chain operate-on-gie-id = 1 # process PGIE objects operate-on-class-ids = 0 # vehicles only (class 0) # Model model-engine-file = color_classifier.engine labelfile-path = color_labels.txt gpu-id = 0 # Preprocessing — ImageNet normalization (must match training) net-scale-factor = 0.0039215697906911373 offsets = 123.675;116.280;103.530 # ImageNet mean×255 color-format = 0 # RGB infer-dims = 3;224;224 # Quality gate — skip tiny/distant vehicles input-object-min-width = 64 input-object-min-height = 64 # Classifier config num-detected-classes = 15 classifier-threshold = 0.2 classifier-async-mode = 0 # sync: results in same frame # Raw tensor output — full softmax[15] for custom stabilization output-tensor-meta = 1 # disables NvDsClassifierMeta network-mode = 2 # FP16 batch-size = 16
EX SGIE Example — License Plate Detector + Char Classifier Chain Example
config_sgie_lp_detector.txtSGIE-3 · LP Detector
[property] gie-unique-id = 3 process-mode = 2 network-type = 0 # 0 = detector (finds LP bbox) operate-on-gie-id = 1 # run on PGIE vehicle objects operate-on-class-ids = 0 model-engine-file = lp_detector.engine gpu-id = 0 net-scale-factor = 0.0039215697906911373 infer-dims = 3;640;640 num-detected-classes = 1 # plate only threshold = 0.50 cluster-mode = 2 nms-iou-threshold = 0.40 input-object-min-width = 80 output-tensor-meta = 1 network-mode = 2 batch-size = 8
config_sgie_lp_classifier.txtSGIE-4 · LP Character Classifier
[property] gie-unique-id = 4 process-mode = 2 network-type = 1 # 1 = classifier operate-on-gie-id = 3 # operate on LP detector objects! operate-on-class-ids = 0 # plate class model-engine-file = lp_char_classifier.engine labelfile-path = char_labels.txt # 0-9 A-Z = 36 classes gpu-id = 0 net-scale-factor = 0.0039215697906911373 color-format = 2 # GRAY — LP images are grayscale infer-dims = 1;32;128 # C;H;W for LP crop num-detected-classes = 36 classifier-threshold = 0.30 classifier-async-mode = 0 output-tensor-meta = 0 # label mode → NvDsClassifierMeta input-object-min-width = 20 network-mode = 2 batch-size = 32 # many LP crops per batch
Chain Rule SGIE-4 uses operate-on-gie-id=3 — it operates on the LP bboxes found by SGIE-3, not on the vehicle bboxes from the PGIE. This is how multi-stage SGIE chains work: each SGIE can operate on any upstream GIE's output objects.
